# Simulation – Simulierbarkeit ≠ Ersetzbarkeit/Identität

## Core Claim / Pattern
Eine Simulation bildet ausgewählte Strukturen/Relationen eines Systems ab, oft mit idealisierten Annahmen.

## Frequent Overinterpretation
‚Perfekte Simulation‘ wird als ‚das simulierte Ding selbst‘ verstanden (z. B. Simulation = Bewusstsein).

## Where the Interpretive Transfer Occurs
Von struktureller Isomorphie/Approximation wird auf ontologische Identität geschlossen.

## Philosophical Residue
Simulationen sind Repräsentationen; Identität erfordert Kriterien (Träger, Kausalrollen, Materialität, Bewusstsein etc.).

## Why the Misinterpretation Is Tempting
Weil erfolgreiche Simulation subjektiv wie das Original wirkt (VR/Agentenverhalten).

## Useful Checks
- **Bridge principle?** Welche zusätzliche Annahme verbindet das formale/epistemische Ergebnis mit einer Aussage über die Wirklichkeit?
- **Scope?** Gilt der Satz über *Modelle/Begriffe/Methoden* oder über *Sein*?
- **Counterfactual:** Was müsste sich ändern, damit das Resultat nicht mehr gilt — die Welt oder das Rahmenwerk?
